# Code of Conduct

`episteme` is a solo-maintained project. The rules below are short on
purpose. They cover what I actually care about in interactions on issues,
PRs, and discussions.

## The two things that matter

1. **Critique ideas, not people.** "This approach is wrong because X" is
   welcome. "You are wrong" / "you don't understand" / personal attacks
   are not. The kernel itself codifies this — `kernel/CONSTITUTION.md`
   Collaboration Stance: *prefer direct critique of ideas, not people.*
2. **Argue from evidence.** If you claim something is broken, name the
   commit, the file, the test, or the observed behavior. If you propose
   a change, name what would prove the change wrong. The whole project
   is about not letting confident-sounding answers pass for examined
   ones — the same standard applies to discussion of the project.

## Specifically welcome

- Bug reports with reproduction steps.
- Disagreement with design choices, framed as: "this would be better if
  X because Y, and Z would prove me wrong."
- Questions about the kernel's claims, including challenges to the
  central empirical thesis. The project is falsifiable on purpose; that
  invites scrutiny.
- "I tried it and it doesn't fit my workflow because…" with details.

## Specifically not welcome

- Personal attacks, harassment, doxing, hate speech, or sexualized
  comments toward anyone.
- Bad-faith engagement — posting to provoke, ignoring questions you
  asked, refusing to engage with disconfirmation.
- Spam, off-topic promotion, or attempts to weaponize issues for
  unrelated agendas.

## Enforcement

I (the maintainer) decide. The escalation is simple:

1. Comment removed and a short note explaining why.
2. Repeated → temporary block from issues/PRs.
3. Severe or pattern → permanent block.

There is no committee. There is no anonymized reporting flow. If you
need to report something privately rather than in an issue, email
[junseong.lee652@gmail.com](mailto:junseong.lee652@gmail.com) with
"episteme conduct" in the subject line.

## Scope

This applies to the GitHub repo (issues, PRs, discussions, commit
comments) and to anywhere I'm representing the project (e.g., posts
linking to this repo). It does not extend to general internet conduct
or to other projects.
