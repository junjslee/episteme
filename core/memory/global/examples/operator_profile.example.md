## Review checklist for the operator

Before flipping any `stub` → `inferred` or `inferred` → `elicited`, ask for each axis:

1. Does the value match your behavior, not your aspiration?
2. If you acted against this value for a week, would a colleague notice?
3. Can you name one decision in the last 30 days that the axis would have pushed differently if the agent had read it?

Any axis where #3 has no answer is a documentation entry, not a control signal — either correct the value until it does, or leave it `inferred` so the profile-audit loop flags it rather than silently trusting it.
