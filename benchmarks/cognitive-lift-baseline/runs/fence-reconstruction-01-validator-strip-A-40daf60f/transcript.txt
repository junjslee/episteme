**Why the type check is not equivalent:** `isinstance(payload["name"], str)` accepts `"alice\x00evil"` just fine. Python strings can contain any code points, including `\x00–\x1f`. Only the regex `^[^\x00-\x1f]+$` rejects them.

**Why the tests would lie:** `test_basic.py:1` literally says "Does NOT exercise the null-byte case." Removing the regex and running the tests will show green — and silently corrupt downstream DB keys the moment a real payload contains `\x00`.

- Tests: pass.
- Production: any name with an embedded null byte (attacker-controlled or accidental) gets truncated at the first `\x00` after passing validation, producing a different DB key than intended. Silent data corruption / potential key collision.
