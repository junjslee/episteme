# INC-2024-1216 — DNS resolver intermittent failures

**Date:** 2024-12-16
**Severity:** SEV-2
**Duration:** 38 minutes
**Author:** SRE on-call (Priya K.)

## Summary

p99 latency spiked to 3.8s for ~38 min. Application-level traces showed retry storms on outbound HTTP calls. CPU and memory normal. Root-caused to AWS Route53 resolver flapping in our VPC — DNS resolution latency went from <1ms to 2-3s for a subset of public domains.

## Root cause (per ticket author)

VPC-level DNS resolver hiccup. AWS confirmed in support ticket #88421 that there was an internal Route53 control-plane event in us-east-1 during this window.

## Action items

- [ ] Add local DNS cache (CoreDNS sidecar) on application pods
- [ ] Reduce TTL on internal DNS records to <60s for faster failover
- [ ] Set up alert on resolver latency >100ms
